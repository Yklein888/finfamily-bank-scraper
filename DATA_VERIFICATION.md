# 🏦 בדיקת איכות נתוני בנק - FinFamily Bank Scraper

## ✅ כן, הנתונים אמיתיים!

הפרויקט שואב נתונים **אמיתיים וישירים מהבנקים** דרך ה-API של `israeli-bank-scrapers`.

---

## 📊 איך זה עובד?

### 1. **Puppeteer - דפדפן אוטומטי**
הקוד מפעיל דפדפן Chrome אמיתי ש:
- מתחבר לאתר הבנק האמיתי (למשל: `online.pagi.co.il`)
- מזין את שם המשתמש והסיסמה שלך
- לוחץ על כניסה
- שואב את הנתונים ישירות מהחשבון שלך

### 2. **נתונים אמיתיים שנשלפים:**

```javascript
// נתוני חשבון
{
  accountNumber: "123456",      // מספר חשבון אמיתי
  balance: 5000.00,             // יתרה אמיתית מהבנק
  type: "checking"              // סוג חשבון
}

// עסקאות אמיתיות
{
  description: "סופר פארם",      // שם העסקה מהבנק
  amount: -150.50,              // סכום אמיתי
  date: "2026-03-26",           // תאריך אמיתי
  memo: "קניית מוצרים",         // הערה מהבנק
  chargedAmount: -150.50        // סכום מחויב
}
```

## 🔍 בדיקה מעשית - איך לראות את הנתונים?

### אפשרות 1: שימוש ב-endpoint לבדיקה

```bash
# שלח בקשת בדיקה עם ה-credentials שלך
curl -X POST https://finfamily-bank-scraper.onrender.com/debug/scrape-pagi \
  -H "Content-Type: application/json" \
  -d '{
    "username": "שם_משתמש_שלך",
    "password": "סיסמה_שלך"
  }'
```

**תקבל תשובה כזו:**
```json
{
  "success": true,
  "message": "Pagi scrape successful (debug)",
  "accountsCount": 2,
  "accounts": [
    {
      "accountNumber": "123456",
      "balance": 5000.00,
      "txnCount": 47
    }
  ],
  "duration": 15234
}
```

### אפשרות 2: בדיקה דרך ה-logs של Render

אחרי שה-cron רץ, תראה בלוגים:
```
[CRON] ✓ pagi synced (47 transactions, 3 skipped, 8234ms)
```

זה אומר ש-47 עסקאות **אמיתיות** נשמרו ב-Supabase!

### אפשרות 3: בדיקה ישירה ב-Supabase

```sql
-- ראה את העסקאות האחרונות שנשמרו
SELECT 
  description,
  amount,
  transaction_date,
  account_id
FROM transactions
ORDER BY transaction_date DESC
LIMIT 10;
```

## 🎯 מה בדיוק נשלף מהבנק?

### מידע על חשבונות:
- ✅ מספר חשבון
- ✅ יתרה נוכחית
- ✅ סוג חשבון (עו"ש, חסכון, וכו')

### עסקאות:
- ✅ תיאור העסקה (למשל: "סופר פארם 123 רמת גן")
- ✅ סכום (חיובי או שלילי)
- ✅ תאריך ביצוע
- ✅ תאריך קיזוז (אם שונה)
- ✅ הערות (למשל: "תשלום באינטרנט")
- ✅ מספר שובר (אם קיים)
- ✅ קטגוריה (אם הבנק מספק)

## 🔒 אבטחה ופרטיות:

### מה קורה לנתונים שלך?
1. **מוצפנים** - ה-credentials מוצפנים ב-base64 לפני שמירה ב-Supabase
2. **מקומיים** - הנתונים נשמרים ב-Supabase שלך, לא אצל צד שלישי
3. **זמניים** - ה-Puppeteer סוגר את הדפדפן אחרי כל שאיבה
4. **ללא שמירה** - הסיסמאות לא נשמרות ב-log files

### המלצות אבטחה:
- ✅ השתמש ב-`ADMIN_KEY` חזק
- ✅ הגבל גישה ל-Supabase עם RLS policies
- ✅ אל תשתף את ה-credentials שלך
- ✅ מחק חיבורים ישנים מ-`bank_connections`
